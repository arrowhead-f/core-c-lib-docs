ah_udp_sock_cbs Struct Reference

Detailed Description

UDP socket callback set.

A set of function pointers used to handle events on UDP sockets, such as a socket being opened or a datagram being received.

Every function takes a context pointer as its first argument. This context pointer comes from the ah_udp_sock_obs owning this callback set. More specifically, it is a copy of the value of the ah_udp_sock_obs::ctx field of the owning ah_udp_sock_obs instance. The context pointer makes it possible for you to associate arbitrary state with individual UDP sockets.

Parameters
ctxPointer to context.
sockPointer to socket.

  • AH_ENONE - Socket opened successfully.
  • AH_EACCES [Darwin, Linux] - Not permitted to open socket.
  • AH_EADDRINUSE - Specified local address already in use.
  • AH_EADDRNOTAVAIL - No available local network interface is associated with the given local address.
  • AH_EAFNOSUPPORT - Specified IP version not supported.
  • AH_ECANCELED - Socket event loop is shutting down.
  • AH_EMFILE [Darwin, Linux, Win32] - Process descriptor table is full.
  • AH_ENETDOWN [Win32] - The network subsystem has failed.
  • AH_ENFILE [Darwin, Linux] - System file table is full.
  • AH_ENOBUFS [Darwin, Linux, Win32] - Not enough buffer space available.
  • AH_ENOMEM [Darwin, Linux] - Not enough heap memory available.
  • AH_EPROVIDERFAILEDINIT [Win32] - Network service failed to initialize.
Note
Every successfully opened sock must eventually be provided to ah_udp_sock_close().

void(* ah_udp_sock_cbs::on_recv) (void *ctx, ah_udp_sock_t *sock, ah_udp_in_t *in, ah_err_t err)

sock has received data.

Successful calls to this function (meaning that err is equal to AH_ENONE) always carry a pointer to an ah_udp_in instance. That instance is reset and reused by sock every time this callback is invoked. If you wish to save the contents of in without having to copy it over to another buffer, you can detach it from sock using ah_udp_in_detach(), which automatically allocates a new input buffer for sock.

If this callback is invoked with an error code (err is not equal to AH_ENONE), sock should always be closed via a call to ah_udp_sock_close().

Parameters
ctxPointer to context.
sockPointer to socket.
inPointer to input data representation, or NULL if err is not AH_ENONE.
errOne of the following codes:
  • AH_ENONE - Data received successfully.
  • AH_ECANCELED - Socket event loop is shutting down.
  • AH_ECONNRESET [Win32] - A previous send operation resulted in an ICMP "Port Unreachable" message.
  • AH_EEOF - Socket closed.
  • AH_EMSGSIZE [Win32] - Received message too large to fit inside in.
  • AH_ENETDOWN [Win32] - The network subsystem has failed.
  • AH_ENETRESET [Win32] - Time to live is enabled for the socket and a related failure was detected.
  • AH_ENOBUFS [Darwin, Linux] - Not enough buffer space available.
  • AH_ENOMEM [Darwin, Linux] - Not enough heap memory available.

◆ on_send

void(* ah_udp_sock_cbs::on_send) (void *ctx, ah_udp_sock_t *sock, ah_udp_out_t *out, ah_err_t err)

sock has sent data.

This callback is always invoked after a successful call to ah_udp_sock_write(). If err is AH_ENONE, all outgoing data provided to the mentioned function was transmitted successfully. If err has any other value, an error occurred before the transmission could be completed. If an error has occurred, sock should be closed using ah_udp_sock_close().

Parameters
ctxPointer to context.
sockPointer to socket.
outPointer to output buffer provided to ah_udp_sock_write(), or NULL if err is not AH_ENONE.
errOne of the following codes:
  • AH_ENONE - Data sent successfully.
  • AH_ECANCELED - Socket event loop is shutting down.
  • AH_ECONNRESET [Win32] - A previous send operation resulted in an ICMP "Port Unreachable" message.
  • AH_EEOF - Socket closed.
  • AH_EHOSTUNREACH [Darwin, Linux] - The targeted remote host could not be reached.
  • AH_EMSGSIZE - Sent message size exceeds supported maximum.
  • AH_ENETDOWN [Darwin, Linux] - The local network interface required to send the message is down.
  • AH_ENETDOWN [Win32] - The network subsystem has failed.
  • AH_ENETRESET [Win32] - Time to live is enabled for the socket and a related failure was detected.
  • AH_ENETUNREACH [Darwin] - Network of remote host not reachable.
  • AH_ENOBUFS [Darwin, Linux] - Not enough buffer space available.
  • AH_ENOMEM [Darwin, Linux] - Not enough heap memory available.
