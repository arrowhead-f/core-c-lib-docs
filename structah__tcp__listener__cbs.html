ah_tcp_listener_cbs Struct Reference

Detailed Description

TCP listener callback set.

A set of function pointers used to handle TCP listener events, such as a listener being opened or a connection being accepted.

Every function takes a context pointer as its first argument. This context pointer comes from the ah_tcp_listener_obs owning this callback set. More specifically, it is a copy of the value of the ah_tcp_listener_obs::ctx field of the owning ah_tcp_listener_obs instance. The context pointer makes it possible for you to associate arbitrary state with individual TCP listeners.

void(* ah_tcp_listener_cbs::on_accept) (void *ctx, ah_tcp_listener_t *ln, ah_tcp_accept_t *accept, ah_err_t err)

ln has accepted the connection in accept.

If err is AH_ENONE, which indicates a successful acceptance, you must

  1. set the connection observer (ah_tcp_accept::obs) in accept, and
  2. call ah_tcp_conn_close() with the connection (ah_tcp_accept::conn) in accept once it is no longer in use.

If you close the connection already in this callback, setting the connection observer in accept becomes unnecessary. If you do not close the connection and also do not set the connection observer in accept, this callback will be invoked again with err set to AH_ESTATE.

The remote address pointer (ah_tcp_accept::raddr) in accept, as well as the accept instance itself, are only guaranteed to exist until this callback returns. You may copy the remote address somewhere else to store it, or you may call ah_tcp_conn_get_raddr() at some later point to retrieve it again.

Parameters
ctxPointer to context.
lnPointer to listener.
acceptPointer to structure containing pointer to the accepted connection and other associated data, or NULL if err is not AH_ENONE.
errAH_ENONE if ln accepted connection successfully. What other error codes are possible depend on the used TCP transport. The following codes may be provided if the default transport is used, directly or indirectly:
  • AH_ECANCELED - Listener event loop is shutting down.
  • AH_ECONNABORTED [Darwin, Linux] - Connection aborted before finalization.
  • AH_ECONNRESET [Win32] - Connection aborted before finalization.
  • AH_EMFILE [Darwin, Linux, Win32] - Process descriptor table is full.
  • AH_ENETDOWN [Win32] - The network subsystem has failed.
  • AH_ENFILE [Darwin, Linux] - System file table is full.
  • AH_ENOBUFS [Linux, Win32] - Not enough buffer space available.
  • AH_ENOMEM [Darwin, Linux] - Not enough heap memory available.
  • AH_EPROVIDERFAILEDINIT [Win32] - Network service failed to initialize.
  • AH_ESTATE - obs not set in a previous call to this function, as explained above.
Note
Every successfully accepted TCP connection must eventually be provided to ah_tcp_conn_close().
Data receiving is disabled for accepted connections by default. It must be explicitly enabled via a call to ah_tcp_conn_read_start().

Parameters

connPointer to listener.
errShould always be AH_ENONE. Other codes may be provided if an unexpected platform error occurs.
Note
This function is guaranteed to be called after every call to ah_tcp_listener_close(), which makes it an excellent place to release any resources associated with ln. You may, for example, elect to call ah_tcp_listener_term() in this callback.

void(* ah_tcp_listener_cbs::on_listen) (void *ctx, ah_tcp_listener_t *ln, ah_err_t err)

ln has started to listen for incoming connections, or the attempt failed.

Parameters
ctxPointer to context.
lnPointer to listener.
errAH_ENONE if ln started to listen successfully. What other error codes are possible depend on the used TCP transport. The following codes may be provided if the default transport is used, directly or indirectly:
  • AH_EACCES [Darwin] - Not permitted to listen.
  • AH_EADDRINUSE [Linux, Win32] - No ephemeral TCP port is available. This error can only occur if the listener was opened with the wildcard address, which means that network interface binding is delayed until listening.
  • AH_ECANCELED - Listener event loop is shutting down.
  • AH_ENETDOWN [Win32] - The network subsystem has failed.
  • AH_ENFILE [Win32] - System file table is full.
  • AH_ENOBUFS [Win32] - Not enough buffer space available.
