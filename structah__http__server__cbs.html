ah_http_server_cbs Struct Reference

Detailed Description

HTTP server callback set.

A set of function pointers used to handle events on HTTP servers.

Every function takes a context pointer as its first argument. This context pointer comes from the ah_http_server_obs owning this callback set. More specifically, it is a copy of the value of the ah_http_server_obs::ctx field of the owning ah_http_server_obs instance. The context pointer makes it possible for you to associate arbitrary state with individual HTTP servers.

void(* ah_http_server_cbs::on_accept) (void *ctx, ah_http_server_t *srv, ah_http_client_t *client, ah_http_client_obs_t *obs, ah_err_t err)

srv has accepted the cln cln.

If err is AH_ENONE, which indicates a successful acceptance, all further events related to cln will be dealt with via the cln callback set (see ah_http_client_cbs) provided when listening was started via ah_http_server_listen().

Parameters
ctxPointer to context.
srvPointer to listener.
clnPointer to accepted cln, or NULL if err is not AH_ENONE.
raddrPointer to address of cln, or NULL if err is not AH_ENONE.
errOne of the following codes:
  • AH_ENONE - Client accepted successfully.
  • AH_ECANCELED - Server event loop is shutting down.
  • AH_ECONNABORTED [Darwin, Linux] - Connection aborted before finalization.
  • AH_ECONNRESET [Win32] - Connection aborted before finalization.
  • AH_EMFILE [Darwin, Linux, Win32] - Process descriptor table is full.
  • AH_ENETDOWN [Win32] - The network subsystem has failed.
  • AH_ENFILE [Darwin, Linux] - System file table is full.
  • AH_ENOBUFS [Linux, Win32] - Not enough buffer space available.
  • AH_ENOMEM [Darwin, Linux] - Not enough heap memory available.
  • AH_EPROVIDERFAILEDINIT [Win32] - Network service failed to initialize.
Note
Every successfully accepted cln must eventually be provided to ah_http_client_close().
In contrast to plain TCP connections, data receiving is always enabled for new HTTP connections.

void(* ah_http_server_cbs::on_close) (void *ctx, ah_http_server_t *srv, ah_err_t err)

srv has been closed.

Parameters
ctxPointer to context.
srvPointer to server.
errShould always be AH_ENONE. Other codes may be provided if an unexpected platform error occurs.
Note
This function is guaranteed to be called after every call to ah_http_server_close(), which makes it an excellent place to release any resources associated with srv. You may, for example, elect to call ah_http_server_term() in this callback.

void(* ah_http_server_cbs::on_listen) (void *ctx, ah_http_server_t *srv, ah_err_t err)

srv has started to listen for connecting clients, or the attempt failed.

Parameters
ctxPointer to context.
srvPointer to server.
errOne of the following codes:
  • AH_ENONE - Server started to listen successfully.
  • AH_EACCES [Darwin] - Not permitted to listen for TCP connections.
  • AH_EADDRINUSE [Linux, Win32] - No ephemeral TCP port is available. This error can only occur if the server was opened with the wildcard address, which means that network interface binding is delayed until listening.
  • AH_ECANCELED - Server event loop is shutting down.
  • AH_ENETDOWN [Win32] - The network subsystem has failed.
  • AH_ENFILE [Win32] - System file table is full.
  • AH_ENOBUFS [Win32] - Not enough buffer space available.
