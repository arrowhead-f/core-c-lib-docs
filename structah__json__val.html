const char* ah_json_val::base

Pointer to position within JSON input where the current value begins.

If type is AH_JSON_TYPE_OBJECT or AH_JSON_TYPE_ARRAY, this pointer is guaranteed to point directly at a { or [ character, respectively. If type has any other value, the length of this string is indicated by length.

Examples
ah_json/examples/interpret_object.c.

◆ length

uintX_t ah_json_val::length

The length of the string referred to by base, or the number of child nodes if type is AH_JSON_TYPE_OBJECT or AH_JSON_TYPE_ARRAY.

When type is AH_JSON_TYPE_OBJECT, the value of this field indicates the total number of keys and values in this object. When type is AH_JSON_TYPE_ARRAY, it indicates the total number of elements in this array.

Note
The concrete type used to represent the current level varies across supported platforms. That type is, however, guaranteed to be able to hold any integer in the range [0, AH_JSON_LENGTH_MAX].
Examples
ah_json/examples/interpret_object.c.

◆ level

uintX_t ah_json_val::level

The tree depth at which this value is located in its input string.

To better understand what a level is, consider the following JSON object:

{
"a": [1, 2, 3],
"b": null
}

Its root object, whose start is indicated by {, is located at level 0. Its two keys, "a" and "b", and their values, the array indicated by [ and null, are at level 1. The numbers 1, 2 and 3, all inside the array, are all at level 2.

When iterating through tokens, you can reliably determine when, for example, an array ends, by looking for a change in the current level.

Note
The concrete type used to represent the current level varies across supported platforms. That type is, however, guaranteed to be able to hold any integer in the range [0, AH_JSON_LEVEL_MAX].
Examples
ah_json/examples/interpret_object.c.
