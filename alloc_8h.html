alloc.h File Reference

Detailed Description

Dynamic memory allocation.

This file most significantly provides function for allocating and freeing dynamic memory. It provides two allocators, a page allocator and the C99 allocator. The implementations of the allocators can be changed by specifying their replacements as directed in conf.h.

Page Allocator

A page is a contiguous chunk of AH_PSIZE bytes, aligned to a memory boundary deemed suitable by the current library platform implementation. The page allocator provides and recycles such pages via the ah_palloc() and ah_pfree() functions.

In contrast to the C99 allocator, the page allocator is relatively simple to implement. As every page is of the exact same size, the allocator may return any of them when new memory is requested. An implementation of a page allocator may, for example, use a predefined set of chunks and keep track of which of them are available for allocation via a single linked list.

The page allocator is used, when possible, by the Arrowhead Core C libraries.

C99 Allocator

The C99 allocator concretely consists of the malloc(), calloc(), realloc() and free() functions. However, implementing these in a manner that is both space-efficient and performant can be difficult, and especially so on embedded platforms. As a consequence, this library, and most other official Arrowhead C libraries, are designed to work even when the implementations of these functions are very limited. More specifically, an acceptable C99 allocator implementation

  1. returns valid pointers when allocating new memory (if the requested number of bytes of heap memory are available),
  2. fails every attempt to reallocate memory (i.e. the first argument to realloc() is not NULL), and
  3. ignores any attempt to free memory.

All library functionality that relies on being able to continually allocate and release memory must use the page allocator or be able to operate without using the C99 allocator at all.

Note
Please refer to conf.h for more details on how to configure what C99 allocator function implementations are to be used.

#define ah_realloc (   ptr,
  size 
)    AH_CONF_REALLOC((ptr), (size))

Reallocates memory chunk associated with ptr.

Conceptually, this function allocates a new memory chunk of size bytes, copies over the contents of the ptr buffer (or as much of it as fits in the new chunk), and then frees the memory associated with ptr. If size is larger than the previous size of the ptr chunk, the additional memory is uninitialized. Practically, this function may expand or contract the ptr chunk in place, or do something else with the same observable result.

Parameters
ptrPointer previously acquired from ah_calloc(), ah_malloc() or ah_realloc().
sizeDesired size, in bytes, of chunk after reallocation.
Returns
A chunk of memory of at least size bytes that contain the contents previously associated with ptr. NULL if the reallocation failed, in which case the chunk of ptr remains unmodified.
Note
If size is not 0 and ptr is NULL, this function will behave exactly as a call to ah_malloc() with the same size.
Warning
If size is not 0 and ptr is not NULL, this function may, if implemented so, always return NULL.
If size is 0 and ptr is not NULL, the function may either return a pointer to a memory chunk with size 0 or return NULL.
If size is 0 and ptr is NULL, the result of calling this function is undefined.
