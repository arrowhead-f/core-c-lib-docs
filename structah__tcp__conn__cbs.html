ah_tcp_conn_cbs Struct Reference

Detailed Description

TCP connection callback set.

A set of function pointers used to handle TCP connection events, such as data being received or a connection being opened.

Every function takes a context pointer as its first argument. This context pointer comes from the ah_tcp_conn_obs owning this callback set. More specifically, it is a copy of the value of the ah_tcp_conn_obs::ctx field of the owning ah_tcp_conn_obs instance. The context pointer makes it possible for you to associate arbitrary state with individual TCP connections.

void(* ah_tcp_conn_cbs::on_connect) (void *ctx, ah_tcp_conn_t *conn, ah_err_t err)

conn has been established to a specified remote host, or the attempt to establish it has failed.

Parameters
ctxPointer to context.
connPointer to connection.
errAH_ENONE if a connection was established successfully. What other error codes are possible depend on the used TCP transport. The following codes may be provided if the default transport is used, directly or indirectly:
  • AH_EADDRINUSE [Darwin, Linux, Win32] - Failed to bind a concrete local address. This error only occurs if the connection was opened with the wildcard address, which means that network interface binding is delayed until connection.
  • AH_EADDRNOTAVAIL [Darwin, Win32] - The specified remote address is invalid.
  • AH_EADDRNOTAVAIL [Linux] - No ephemeral TCP port is available.
  • AH_EAFNOSUPPORT - The IP version of the specified remote address does not match that of the bound local address.
  • AH_ECANCELED - The event loop of conn has shut down.
  • AH_ECONNREFUSED - Connection attempt ignored or rejected by targeted remote host.
  • AH_ECONNRESET [Darwin] - Connection attempt reset by targeted remote host.
  • AH_EHOSTUNREACH - The targeted remote host could not be reached.
  • AH_ENETDOWN [Darwin] - Local network not online.
  • AH_ENETDOWN [Win32] - The network subsystem has failed.
  • AH_ENETUNREACH - Network of targeted remote host not reachable.
  • AH_ENOBUFS - Not enough buffer space available.
  • AH_ENOMEM - Not enough heap memory available.
  • AH_ETIMEDOUT - The connection attempt did not complete before its deadline.
Note
Data receiving is disabled for new connections by default. It must be explicitly enabled via a call to ah_tcp_conn_read_start().
This function is never called for accepted connections, which means it may be set to NULL when this data structure is used with ah_tcp_listener_listen().

void(* ah_tcp_conn_cbs::on_read) (void *ctx, ah_tcp_conn_t *conn, ah_tcp_in_t *in, ah_err_t err)

conn has received data from its associated remote host.

Successful calls to this function (meaning that err is equal to AH_ENONE) always carry a pointer to an ah_tcp_in instance. That instance is reused by conn every time this callback is invoked. If the ah_rw field of that instance is not read in its entirety, whatever unread contents remain when this callback returns will be presented again in another call to this callback. If not all of the contents of in are read or discarded every time this callback is invoked, or the buffer is repackaged via ah_tcp_in_repackage(), that buffer may eventually become full, triggering the AH_EOVERFLOW error. If you wish to save the contents of in without having to copy it over to another buffer, you can detach it from conn using ah_tcp_in_detach(), which allocates a new input buffer for conn.

If this callback is invoked with an error code (err is not equal to AH_ENONE), conn should always be closed via a call to ah_tcp_conn_close().

Parameters
ctxPointer to context.
connPointer to connection.
inPointer to input data representation, or NULL if err is not AH_ENONE.
errAH_ENONE if a data was received successfully. What other error codes are possible depend on the used TCP transport. The following codes may be provided if the default transport is used, directly or indirectly:
  • AH_ECANCELED - Connection event loop is shutting down.
  • AH_ECONNABORTED [Win32] - Virtual circuit terminated due to time-out or other failure.
  • AH_ECONNRESET [Darwin, Win32] - Connection reset by remote host.
  • AH_EDISCON [Win32] - Connection gracefully closed by remote host.
  • AH_EEOF - Connection closed for reading.
  • AH_ENETDOWN [Win32] - The network subsystem has failed.
  • AH_ENETRESET [Win32] - Keep-alive is enabled for the connection and a related failure was detected.
  • AH_ENOBUFS [Darwin, Linux] - Not enough buffer space available.
  • AH_ENOMEM [Linux] - Not enough heap memory available.
  • AH_EOVERFLOW - The input buffer of conn is full. Note that the input buffer is not available via in if this error code is provided. The only way to recover from this error is by closing the connection. To prevent this error from occurring, you must ensure that the input buffer never gets exhausted by reading, discarding, repackaging or detaching its contents, as described further above.
  • AH_ETIMEDOUT - Connection timed out.

void(* ah_tcp_conn_cbs::on_write) (void *ctx, ah_tcp_conn_t *conn, ah_tcp_out_t *out, ah_err_t err)

conn has sent data to its associated remote host.

This callback is always invoked after a successful call to ah_tcp_conn_write(). If err is AH_ENONE, all outgoing data provided to the mentioned function was transmitted successfully. If err has any other value, an error occurred before the transmission could be completed. If an error has occurred, conn should be closed using ah_tcp_conn_close().

Parameters
ctxPointer to context.
connPointer to connection.
outPointer to output buffer provided to ah_tcp_conn_write(). If err is not AH_ENONE, it may, but should never, be NULL.
errAH_ENONE if the data was sent successfully. What other error codes are possible depend on the used TCP transport. The following codes may be provided if the default transport is used, directly or indirectly:
  • AH_ECANCELED - Connection event loop is shutting down.
  • AH_ECONNABORTED [Win32] - Virtual circuit terminated due to time-out or other failure.
  • AH_ECONNRESET [Darwin, Linux, Win32] - Connection reset by remote host.
  • AH_EEOF - Connection closed for writing.
  • AH_ENETDOWN [Darwin] - Local network not online.
  • AH_ENETDOWN [Win32] - The network subsystem has failed.
  • AH_ENETRESET [Win32] - Keep-alive is enabled for the connection and a related failure was detected.
  • AH_ENETUNREACH [Darwin] - Network of remote host not reachable.
  • AH_ENOBUFS [Darwin, Linux, Win32] - Not enough buffer space available.
  • AH_ENOMEM [Darwin, Linux] - Not enough heap memory available.
  • AH_ETIMEDOUT - Connection timed out.
