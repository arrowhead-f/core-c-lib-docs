ah_extern ah_err_t ah_utf8_from_codepoint ( uint32_t  codepoint,
char *  dst,
size_t *  dst_length 
)

Writes the UTF-8 sequence representing codepoint to dst.

If the operation is successful, the value pointed at by dst_length is updated to reflect the final length of the string actually written to dst.

Parameters
codepointUnicode codepoint.
dstPointer to receiver of UTF-8 sequence.
dst_lengthPointer to size of dst, in bytes.
Returns
One of the following error codes:
  • AH_ENONE - Operation successful.
  • AH_EINVAL - dst is NULL or size is 0.
  • AH_EINVAL - codepoint is invalid. More specifically, it is either between 0xD800 and 0xDFFF or above 0x10FFFF. The former range is reserved for constructing surrogate pairs with UTF-16, while the latter range is simply not defined for UTF-8.
  • AH_EOVERFLOW - UTF-8 sequence too long to fit in dst.
Note
As per RFC3629, no UTF-8 sequence will ever be longer than 4 bytes, even though sequences of up to 6 bytes are theoretically possible.

Validates UTF-8 text in src.

A UTF-8 text is considered valid if it only contains complete UTF-8 sequences and none of those sequences encodes an invalid codepoint.

Parameters
srcPointer to string containing UTF-8 encoded text.
sizeSize of src, in bytes.
Returns
true only if src contains a valid UTF-8 encoded text. false otherwise.
